# 계층적 에이전트 아키텍처 현대화 계획 (Hierarchical Agent Architecture Modernization Plan)

## 1. 개요
본 계획은 최신 업계 표준(LangGraph, CrewAI, AutoGen) 및 상용 서비스 패턴을 바탕으로 "orchagent"의 계층적 에이전트 아키텍처를 현대화하는 방안을 제시합니다. 목표는 수동적이고 동기적인 서브그래프 호출 모델에서 완전히 통합되고 상태 공유가 가능하며 탄력적인 계층 시스템으로 전환하는 것입니다.

## 2. 조사 결과 및 업계 표준
*   **슈퍼바이저 패턴 (Supervisor Pattern):** 전문화된 팀에 작업을 라우팅하는 중앙 "두뇌" 역할.
*   **계층적 검증 (Hierarchical Validation):** 실행 노드와 슈퍼바이저 사이에 "비판자(Critic)" 또는 "검증자(Validator)" 노드를 배치.
*   **공유 워크스페이스 (Shared Workspace):** 단순 메시지 전달을 넘어 공유된 "초안(Draft)" 또는 "문서(Document)" 상태로 발전.
*   **Human-in-the-Loop (HITL):** 고위험 의사결정을 위한 명시적인 검토 단계 도입.
*   **병렬 실행 (Parallel Execution):** 가능한 경우 독립적인 하위 작업을 동시에 실행하여 성능 향상.

## 3. 현재 구현 상태 분석
*   **강점:** 라우팅을 위해 `Command`를 사용하는 현대적인 방식; `agent-core`와 `apps/backend/workflow` 간의 깔끔한 분리.
*   **약점:** 서브그래프가 `.invoke()`를 통해 호출되어 동기적으로 작동하며, 이로 인해 그래프의 실행 컨텍스트가 단절됨(경계 간 "타임 트래블" 및 체크포인팅 기능 상실). 상태 공유가 메시지 전달에 국한됨. 자동화된 검증/비판 루프 부재.

## 4. 제안된 현대화 전략

### Phase 1: 코어 아키텍처 강화
*   **[x] 네이티브 서브그래프 통합 (진행 중):** 수동 `.invoke()` 호출을 LangGraph의 네이티브 서브그래프 지원으로 교체하여 원활한 상태 전파와 체크포인팅을 가능하게 함.
*   **[x] 공유 글로벌 상태:** `BaseAgentState`를 확장하여 팀 경계를 넘어 보존되는 구조화된 데이터(예: 추출된 데이터, 생성된 초안)를 위한 `shared_context` 딕셔너리 추가.
*   **[x] 비동기 실행:** I/O 집약적 작업(웹 스크래핑, API 호출)의 성능 향상을 위해 서브그래프를 비동기 실행 구조로 리팩토링.

### Phase 2: 신뢰성 및 검증
*   **[ ] 비판자/검증자 노드:** 각 `TeamBuilder`에 선택적인 `validator_node`를 추가하여 작업자의 출력물이 원래 작업 요청에 부합하는지 슈퍼바이저 반환 전에 검증.
*   **[ ] 자가 수정 루프 (Self-Correction Loop):** 검증 실패 시 슈퍼바이저가 피드백과 함께 작업을 팀으로 다시 돌려보낼 수 있도록 허용.
*   **[ ] 추론 기반 슈퍼바이저:** 현대적인 모델(예: GPT-5.4)의 "추론(reasoning)" 기능을 슈퍼바이저에서 활용하여 라우팅 전 상세 계획 수립.

### Phase 3: 고급 기능 도입
*   **[ ] Human-in-the-Loop (HITL) 훅:** "검토" 단계를 위해 헤드 슈퍼바이저에 `interrupt_before` 메커니즘 구현.
*   **[ ] 동적 도구 할당:** 작업의 복잡도에 따라 팀이 추가 도구 또는 "하위 에이전트"를 동적으로 요청할 수 있도록 허용.
*   **[ ] 추적성 통합:** 기존 `trace_service`를 깊게 통합하여 "지휘 체계(어떤 슈퍼바이저가 어떤 팀을 호출했는지)" 추적.

## 5. 구현 로드맵
1.  **`agent-core` 리팩토링:** 공유 컨텍스트와 네이티브 서브그래프를 지원하도록 `TeamBuilder` 및 `supervisor.py` 업데이트.
2.  **연구(Research) 및 비전(Vision) 팀 업데이트:** 검증 로직 및 개선된 상태 관리 구현.
3.  **`main_graph` 강화:** HITL 및 글로벌 계획 기능 통합.
4.  **테스트 추가:** 팀 간 계층적 라우팅 및 상태 보존이 예상대로 작동하는지 검증.

## 6. 기대 효과
*   **신뢰성 향상:** 검증 루프를 통해 할루시네이션 감소 및 작업 완수 능력 향상.
*   **성능 최적화:** 병렬 실행 및 비동기 작업을 통한 응답 속도 개선.
*   **확장성:** 새로운 팀이나 전문화된 하위 에이전트를 쉽게 추가할 수 있는 구조.
*   **디버깅 용이성:** LangGraph의 체크포인팅을 통해 전체 계층에 걸친 완벽한 상태 보존 및 추적 가능.
